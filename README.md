# STEP

## 概要

このプロジェクトは、microCMSを利用して低コスト・短期間で構築できるスニーカープロモーション用特設サイトのサンプルです。
非エンジニアでも更新可能な運用を前提とし、バックエンド保守を最小限に抑えた構成を採用しています。

---

## 技術スタック

- HTML / SCSS / JavaScript
- Gulp（ビルドツール）
- microCMS（ヘッドレスCMS）
- Cloudflare Pages（想定ホスティング）

---

## 技術選定の理由

### microCMS

- 非エンジニアでも直感的に更新可能
- APIベースでシンプルな構成
- サーバー管理不要で低コスト運用

### HTML / SCSS / JavaScript

- 短期間での制作を考慮し、一番慣れているスキルを活用。
- 不要な複雑性を排除し、シンプルな構成を優先。

### Gulp

- SCSSコンパイル、圧縮、リロードなどの開発効率を改善
- 既存の設定を使って素早く開発を始められる

### Cloudflare Pages

- 無料で利用可能
- CDNによる高速配信
- サーバーレスで保守不要

---

## システム構成

microCMS（コンテンツ管理）  
↓  
JavaScriptでAPI取得  
↓  
HTMLに描画  
↓  
Cloudflare Pagesで配信 

---

## CMSデータ設計（microCMS）

### API名

`news`

### フィールド構成

| フィールド | 型 | 必須 | 説明 |
|-----------|----|------|------|
| title_ja | テキスト | ◯ | 日本語タイトル |
| title_en | テキスト | ◯ | 英語タイトル |
| body_ja | リッチテキスト | ◯ | 日本語本文 |
| body_en | リッチテキスト | ◯ | 英語本文 |
| thumbnail | 画像 | - | サムネイル |

### 設計意図

多言語対応において、  
**1コンテンツ内で言語ごとのフィールドを管理する設計**を採用しました。

これにより：

- 記事の対応関係を明確に保てる  
- 管理画面で一元管理できる  
- 更新ミスを防止できる  

---

## 多言語対応

- パスで言語を分割：`/ja/`, `/en/`
- フロント側でURLパスを判定して言語表示を切り替え

---

## SEO対応

- URL単位で言語を分離（/ja /en）

---

## ディレクトリ構成

```
src/
  ├ ja/
  ├ en/
  ├ js/
  ├ scss/
  ├ images/

dist/
  ├ ja/
  ├ en/
  └ assets/
```
- src：開発用
- dist：本番配信用

Cloudflare Pagesでは dist をデプロイ対象としています。
---

## 開発手順

```bash
npm install
npx gulp
```

---

## 運用フロー

1. microCMSで記事作成
2. API経由で取得
3. フロントに反映
4. Cloudflare Pagesで公開

---

## インフラ構成

- Cloudflare Pagesを基本としたサーバーレス配信
- CDNにより高速配信
- 無料枠で運用可能
- サーバー保守不要

---

## セキュリティ

- APIキーは公開リポジトリに含めない（環境変数 `MICROCMS_API_KEY` を利用）
- クライアント側で必要最低限のAPI呼び出しのみ実行
- 本番運用ではCloudflare Pagesの環境変数設定を使用
